# 枕下压电 BCG 呼吸信号提取研究方案

## 1. 研究定位

本课题面向枕下压电传感采集的 BCG 混叠信号，目标是从其中提取与呼吸相关的代理表征，用于估计受试者的呼吸节律、呼吸努力变化和睡眠呼吸障碍相关趋势。

在当前数据条件下，任务不宜定义为“从 BCG 精确还原 PSG 胸带/腹带原始波形”，而应定义为：

> 基于弱同步 PSG 参考的枕下压电 BCG 呼吸努力代理信号估计。

模型需要输出的是一个在 BCG 时间轴上可信的呼吸代理信号，以及呼吸率、努力包络、事件风险和置信度等辅助结果。该输出应在窗口级、事件级和频谱级与 PSG 胸带/腹带参考一致，而不是追求逐点相位完全一致。

## 2. 数据前提与核心约束

### 2.1 PSG 与 BCG 未精确同步

两路信号未能获得精确同步，因此不应直接使用逐点波形监督作为主要训练目标。

不推荐的目标：

```text
BCG(t) -> 胸带(t)
BCG(t) -> 腹带(t)
BCG(t) -> RIPsum(t)
```

不推荐的主损失：

```text
MSE(预测呼吸波形, PSG 胸/腹带波形)
```

即使模型预测出了正确的呼吸节律，只要 BCG 与 PSG 存在秒级错位，逐点 MSE 也会错误惩罚模型，导致模型学习出过度平滑、幅值被压缩的平均波形。

### 2.2 体动已在信号处理初期剔除

体动剔除可以降低建模难度，但需要保留以下信息：

- 原始时间戳；
- clean mask；
- 被剔除片段的起止位置；
- clean segment 与原始记录的映射关系；
- 每个事件附近是否存在体动剔除。

体动段最好被视为“不可用片段”，在训练和评价中 mask 掉，而不是直接物理删除后拼接成新的连续信号。若体动段已经被删除，则必须维护片段映射表，否则会进一步破坏 BCG 与 PSG 的时间关系。

### 2.3 单通道枕下信号的可辨识性有限

单通道枕下 BCG 不一定能唯一还原胸带和腹带两路独立运动。更稳妥的第一目标是估计综合呼吸努力或 BCG-derived respiratory effort surrogate。

胸带、腹带分离可以作为后续探索任务，但不应作为第一阶段主目标。

## 3. 研究目标分层

### Level 1：呼吸节律估计

目标：

- 呼吸率轨迹；
- 呼吸周期长度；
- 呼吸主频变化；
- 窗口级呼吸次数。

该层最稳定，适合作为基础目标。

### Level 2：呼吸努力幅值趋势估计

目标：

- 呼吸努力强弱变化；
- 低通气相关幅值下降；
- 呼吸暂停后恢复呼吸增强；
- 周期性呼吸或努力波动；
- 局部努力包络变化。

该层是课题的核心，但应采用相位不敏感监督，重点比较趋势、相对幅值和事件前后变化。

### Level 3：睡眠呼吸障碍事件检测

目标：

- 呼吸暂停/低通气事件风险；
- 事件前后呼吸努力变化；
- 恢复呼吸增强；
- AHI 或严重程度等级估计；
- 有效片段内的事件检测。

下游事件检测不应只依赖预测出的呼吸波形，还应融合原始 BCG 特征、心冲击调制特征、时频特征和质量特征。

## 4. 信号信息利用原则

### 4.1 BCG 是耦合观测，不是简单加性噪声

枕下压电 BCG 可近似理解为以下成分的混合：

```text
呼吸运动 + 心冲击图 + 体动残余 + 接触状态变化 + 机械传递噪声 + 环境噪声
```

这些成分不是完全独立的。呼吸会调制心冲击幅值、基线、形态和能量分布；体位、枕头接触、头颈位置也会改变传递路径。因此，前处理阶段不宜强行做硬分离。

推荐做法是模型内部软分解：

```text
raw BCG
  -> respiratory latent branch
  -> cardiac modulation branch
  -> time-frequency branch
  -> quality/context branch
  -> gated fusion
  -> respiratory surrogate / RR / envelope / event / confidence
```

### 4.2 呼吸分量

呼吸分量是主输出，但应定义为 BCG 时间轴上的呼吸代理信号，而不是 PSG 胸带或腹带的复制品。

建议输出：

- `r_hat(t)`：BCG-derived respiratory surrogate waveform；
- `A_hat(t)`：呼吸努力包络；
- `RR_hat(t)`：呼吸率轨迹；
- `P_breath(t)`：呼吸峰/谷或周期概率；
- `P_event(t)`：呼吸事件概率；
- `Q_hat(t)`：置信度或可用性。

### 4.3 心冲击分量

心冲击不应简单视为噪声。它可能通过以下方式提供呼吸信息：

- 心冲击幅值受呼吸调制；
- 心冲击基线随呼吸运动变化；
- 心搏形态受胸腔压力、体位和接触状态影响；
- 心率和心搏间期可能反映事件后恢复、觉醒或自主神经变化。

建议保留 cardiac branch，用于提取：

- 心冲击时域形态；
- 心冲击频带能量；
- 心冲击包络；
- 心率或心搏间期粗估计；
- 心冲击调制特征。

### 4.4 时频信息

时频图有价值，但不应作为唯一输入，也不应以“修复时频图再反变换”为主路线。

更合理的使用方式：

- 辅助识别呼吸频谱 ridge；
- 辅助观察呼吸频带能量连续性；
- 辅助区分呼吸、心冲击和残余噪声；
- 用窗口级频谱结构监督模型。

时频特征应与时域原始信号、滤波候选带和心冲击包络共同输入模型。

### 4.5 接触状态与低频漂移

低频漂移不一定全部是噪声，也可能反映：

- 头颈位置变化；
- 枕头受力变化；
- 体位变化；
- 传感器接触状态变化；
- 机械传递路径变化。

这类信息不适合作为呼吸主输出，但适合作为质量和上下文特征，用于判断当前片段是否可靠。

## 5. PSG 参考标签构造

由于同步不精确，应优先从 PSG 胸带、腹带构造相位不敏感标签。

### 5.1 综合努力参考

可构造：

```text
RIPsum = normalized(胸带) + normalized(腹带)
```

注意不要只做全局归一化后丢弃幅值变化。低通气、恢复呼吸增强等任务需要保留相对幅值趋势。

### 5.2 推荐标签

从 PSG 中提取：

- `RR_ref[k]`：窗口级呼吸率；
- `A_ref[k]`：窗口级努力幅值；
- `Drop_ref[k]`：相对幅值下降比例；
- `Spec_ref[k, f]`：呼吸频带归一化谱；
- `Ridge_ref[k]`：主呼吸频率；
- `Event_ref[k]`：窗口级呼吸事件标签；
- `Recovery_ref[k]`：恢复呼吸增强标签。

这些标签比原始逐点波形更适合弱同步数据。

### 5.3 同步可信度分级

建议为每一晚或每个 clean segment 建立同步可信度等级。

#### S2：局部同步可信

特征：

- BCG 低频呼吸候选信号与 PSG RIPsum 存在稳定高相关；
- 最佳 lag 较稳定；
- 局部峰谷关系基本可信。

可用监督：

- shift-invariant waveform loss；
- 峰/谷弱监督；
- 包络损失；
- 呼吸率损失；
- 频谱损失。

#### S1：粗同步可信

特征：

- 知道 BCG 与 PSG 属于大致相同时间段；
- 秒级峰谷不可靠；
- 窗口级统计仍可信。

可用监督：

- 呼吸率损失；
- 包络趋势损失；
- 频谱 ridge 损失；
- 窗口级事件损失。

不建议使用逐点波形损失。

#### S0：同步关系不可信

特征：

- 无法确认 BCG 与 PSG 的时间对应关系；
- 局部或窗口级对应都不可靠。

可用方式：

- BCG 自监督预训练；
- record-level 标签；
- 表征学习；
- 整夜严重程度标签。

不应声称从 S0 片段学习到了逐时呼吸努力变化。

## 6. 推荐模型：WeakSync-RespBCG-Net

### 6.1 输入

```text
x_raw       : 原始 BCG clean 片段
x_resp      : BCG 呼吸候选频带
x_card      : BCG 心冲击候选频带
e_card      : 心冲击包络
TF          : STFT / CWT / SST 时频图
q_pre       : rolling energy / STD / derivative energy / clean mask
```

### 6.2 编码器

- 1D Temporal Encoder：学习时域节律、局部形态和长程变化；
- 2D Time-Frequency Encoder：学习呼吸 ridge、频带能量和非平稳变化；
- Cardiac Modulation Encoder：学习心冲击调制信息；
- Quality/Context Encoder：学习接触状态、低频漂移和片段可信度。

### 6.3 融合方式

推荐使用 gated fusion 或 cross-attention。融合模块应允许模型根据片段质量动态调整不同分支的权重。

例如：

```text
raw/time branch + TF branch + cardiac branch + quality branch
                    -> gated fusion
                    -> multi-task heads
```

### 6.4 输出

```text
r_hat(t)     : BCG 时间轴上的呼吸代理波形
A_hat(t)     : 呼吸努力包络
RR_hat(t)    : 呼吸率轨迹
P_breath(t)  : 呼吸峰/谷或周期概率
P_event(t)   : 呼吸事件概率
Q_hat(t)     : 置信度/可用性
```

可选输出：

```text
HR_hat(t)       : 心率轨迹
J_peak_hat(t)   : 心搏位置概率
Card_env_hat(t) : 心冲击包络
```

## 7. 窗口设计

推荐采用多尺度窗口。

### 短窗：8-16 秒

用于：

- 局部波形；
- 心冲击形态；
- 呼吸峰谷；
- 短时质量判断。

### 中窗：30-60 秒

用于：

- 呼吸率估计；
- 呼吸频谱 ridge；
- 窗口级事件标签；
- 包络趋势。

### 长窗：90-180 秒

用于：

- 低通气前后趋势；
- 呼吸暂停后恢复；
- 周期性呼吸；
- 长程努力变化。

训练时不建议只随机切短窗，否则容易丢失事件前后趋势和幅值变化。

## 8. 损失函数设计

总损失可设计为：

```text
L =
  λ1 * L_RR
+ λ2 * L_env
+ λ3 * L_spec
+ λ4 * L_event
+ λ5 * L_shift_wave
+ λ6 * L_smooth
+ λ7 * L_uncertainty
+ λ8 * L_cardiac_aux
```

其中 `L_shift_wave` 只用于同步可信片段，不作为全数据主损失。

### 8.1 呼吸率损失

```text
L_RR = MAE(RR_hat, RR_ref)
```

若存在局部错位，可在受限窗口内使用允许偏移的损失：

```text
L_RR = min over τ MAE(RR_hat(t), RR_ref(t + τ))
```

也可在 RR 轨迹上使用 Soft-DTW，但需要限制 warping window，避免过度扭曲。

### 8.2 包络趋势损失

```text
L_env = 1 - Corr(A_hat, A_ref)
```

或使用相对幅值下降：

```text
Drop = 当前努力幅值 / 局部基线努力幅值
L_drop = Huber(Drop_hat, Drop_ref)
```

该损失适合低通气、努力下降和恢复呼吸增强任务。

### 8.3 频谱结构损失

可比较呼吸频带内的归一化频谱：

```text
L_spec = KL(P_hat(f | window), P_ref(f | window))
```

也可比较主呼吸频率：

```text
L_ridge = MAE(f_resp_hat, f_resp_ref)
```

频谱损失只作为结构约束，不能替代包络、事件和呼吸率监督。

### 8.4 事件损失

若有 PSG 事件标签，可训练窗口级事件概率：

```text
normal / apnea / hypopnea / recovery / low-confidence
```

同步不精确时，事件标签应放宽到 30 秒或 60 秒窗口，不宜要求精确到某一秒。

可选方法：

- BCE；
- focal loss；
- multi-instance learning；
- 带时间容忍窗口的 event loss。

### 8.5 Shift-invariant waveform loss

仅用于 S2 同步可信片段：

```text
L_shift = min over τ [1 - Corr(r_hat(t), y_ref(t + τ))]
```

可在最佳 lag 下附加 Huber 或 L1，但不建议在全部数据上使用。

### 8.6 不确定性损失

模型应输出 `Q_hat(t)` 或不确定性 `s(t)`。可采用：

```text
L = exp(-s) * L_task + s
```

该形式允许模型在低质量片段上降低置信度，同时通过 `+s` 避免模型把所有片段都判为不可信。

### 8.7 心冲击辅助损失

若有 ECG 或可靠心搏参考，可加入：

- HR 损失；
- J 峰检测损失；
- 心搏间期损失。

若没有 ECG，可使用自监督约束：

- 心冲击频带周期性；
- 心搏能量包络平滑性；
- 局部心搏形态一致性。

## 9. GAN 与对比学习的定位

### 9.1 GAN 不宜作为主路线

GAN 容易生成形态上合理但生理上不可靠的呼吸波形。在睡眠呼吸障碍检测中，这种“漂亮但错误”的输出可能掩盖低通气或暂停。

若使用 GAN，应满足：

- 只作为小权重正则项；
- 判别器应 conditional on BCG 和质量特征；
- 主监督仍为 RR、包络、频谱 ridge、事件和置信度；
- 必须用事件级指标验证，而不能只看波形是否平滑。

### 9.2 对比学习适合预训练

对比学习适合在同步不精确、标注有限的条件下学习稳定表征。

可设计正样本：

- 同一 BCG 片段的不同增强版本；
- 同一粗时间窗口内的 BCG 与 PSG 特征；
- 同一受试者同一睡眠阶段的相邻片段；
- 呼吸率或事件标签相近的片段。

可设计增强：

- 幅值缩放；
- 时间裁剪；
- 轻微时间扭曲；
- 噪声注入；
- 频带遮蔽；
- 时频图遮蔽。

对比学习用于 encoder 预训练，最终仍需要 PSG 弱同步标签进行监督微调。

## 10. 下游睡眠呼吸障碍检测

不建议采用单链路：

```text
BCG -> 呼吸波形 -> 事件检测
```

更稳妥的是双路径融合：

```text
路径 A: BCG -> 呼吸代理波形 / 努力包络 / RR -> 事件检测
路径 B: 原始 BCG / 心冲击调制 / 时频特征 / 质量特征 -> 事件检测
融合: 路径 A + 路径 B + clean mask -> 最终事件概率
```

这样可以避免呼吸波形估计错误导致下游全部失败，也能利用心冲击、恢复呼吸、接触状态和质量变化中包含的事件信息。

## 11. 评价指标

### 11.1 呼吸节律指标

- RR MAE；
- RR RMSE；
- breath count error；
- dominant respiratory frequency error。

建议按 30 秒或 60 秒窗口评价。

### 11.2 努力幅值指标

- 包络趋势相关性；
- 幅值下降检测 F1；
- 恢复呼吸增强检测 F1；
- 低通气前后相对幅值误差；
- 周期性呼吸趋势一致性。

### 11.3 事件级指标

- apnea/hypopnea sensitivity；
- precision；
- F1；
- AHI error；
- severity class agreement。

事件时间应允许容忍窗口，例如 ±10 秒、±15 秒或 ±30 秒。

### 11.4 覆盖率指标

由于体动已剔除，必须报告：

- 有效分析时长占比；
- 被剔除片段占比；
- 事件附近被剔除比例；
- 每夜可用数据比例；
- 不同质量等级下的性能。

否则模型可能只在干净片段表现良好，但无法覆盖真实睡眠呼吸障碍事件。

### 11.5 波形指标

波形相关性只能作为辅助指标，并且需要满足：

- 只在 S2 同步可信片段计算；
- 允许最佳 lag 对齐；
- 不作为主要结论依据。

## 12. 实施路线

### Step 1：同步可信度审计

对每晚和每个 clean segment：

- 提取 BCG 低频候选呼吸信号；
- 构造 PSG RIPsum；
- 计算 cross-correlation；
- 估计最佳 lag；
- 统计最大相关系数；
- 观察 lag 是否稳定。

输出：

- 每晚 lag 分布；
- 每段 lag 分布；
- S2/S1/S0 片段比例；
- 是否存在固定 offset、缓慢 drift 或局部同步失效。

### Step 2：构造弱同步标签

从 PSG 胸带/腹带提取：

- `RR_ref`；
- `A_ref`；
- `Drop_ref`；
- `Spec_ref`；
- `Ridge_ref`；
- `Event_ref`；
- `Recovery_ref`。

不先训练逐点波形回归。

### Step 3：建立传统基线

实现并统一评价：

- 低通滤波；
- EWT；
- IEWT；
- CWT/SST ridge tracking；
- Hilbert-based RR；
- 简单包络估计。

传统方法也应使用弱同步评价指标，避免用未对齐波形相关性直接否定其表现。

### Step 4：训练弱同步多任务模型

第一阶段模型输出：

- RR；
- effort envelope；
- spectral ridge；
- event probability；
- confidence。

此阶段不强求 `r_hat(t)` 与 PSG 逐点对齐。

### Step 5：加入呼吸代理波形 head

当 RR、包络和事件趋势稳定后，再加入：

```text
r_hat(t)
```

该输出使用以下约束：

- shift-invariant waveform loss；
- smoothness prior；
- spectral consistency；
- envelope consistency；
- breath peak weak labels。

不要使用全数据逐点 MSE 作为主监督。

### Step 6：融合下游事件检测

使用以下特征共同检测事件：

- `r_hat(t)`；
- `A_hat(t)`；
- `RR_hat(t)`；
- 原始 BCG 特征；
- 心冲击调制特征；
- 时频特征；
- quality / clean mask。

最终评价事件 F1、AHI error、严重程度分类和有效覆盖率。

## 13. 关键风险与控制策略

### 13.1 未精确同步导致错误监督

风险：

- 逐点 MSE 将同步误差误判为模型误差；
- 模型输出被迫平滑化；
- 呼吸峰谷和幅值下降被削弱。

控制：

- 建立同步可信度分级；
- 主用窗口级、频谱级和包络级监督；
- 波形损失只用于 S2 片段。

### 13.2 单通道 BCG 无法唯一还原胸腹两路

风险：

- 模型根据人群先验猜测胸/腹分量；
- 胸腹反常运动无法可靠恢复。

控制：

- 第一目标设为综合努力代理；
- 胸/腹分离作为探索任务；
- 对不确定输出进行显式建模。

### 13.3 时频图修复可能丢失相位和幅值意义

风险：

- 频谱看似合理，但事件边界不可信；
- 反变换后的时域波形缺乏生理解释。

控制：

- 时频图作为辅助输入；
- 输出仍以 RR、包络、事件、置信度和代理波形为主；
- 频谱损失不单独作为主目标。

### 13.4 GAN 幻觉风险

风险：

- 生成正常呼吸样式；
- 掩盖低通气或暂停；
- 使下游事件检测偏乐观。

控制：

- GAN 仅作为弱正则；
- 主监督保持在生理和事件相关目标；
- 用事件级指标验证。

### 13.5 体动剔除导致评价偏差

风险：

- 只在容易片段上评价；
- 事件附近大量剔除导致真实性能被高估。

控制：

- 报告有效覆盖率；
- 报告事件附近剔除比例；
- 按质量等级分层评价。

## 14. 最终建议

本课题的核心主张应从“恢复一个漂亮的呼吸波形”转为：

> 模型能够从枕下压电 BCG 中提取与呼吸相关的代理表征，稳定估计呼吸节律、努力幅值变化、低通气/暂停相关趋势和事件风险；在体动剔除后的有效片段中，这些输出在窗口级、事件级和频谱级与 PSG 胸/腹带参考一致，并优于低通、EWT、IEWT 等传统方法。

推荐主路线：

```text
BCG 多视图输入
  -> 弱同步多任务网络
  -> 呼吸代理波形 / RR / 努力包络 / 事件概率 / 置信度
  -> 下游睡眠呼吸障碍检测
```

推荐模型重点：

- 不以逐点波形还原为核心；
- 不把时频图修复作为唯一目标；
- 不把心冲击简单滤除；
- 不在体动或低可信片段上强行生成呼吸；
- 优先学习相位不敏感的呼吸节律、幅值趋势、频谱结构和事件信息。

这一路线更符合当前数据条件，也更容易形成清晰、可信、可评价的研究结果。

# 枕下压电 BCG 呼吸信号提取研究方案

## 0. 论文研究背景与主线

本研究面向睡眠场景下的枕下压电 BCG 信号，关注其在非接触或弱接触条件下承载呼吸努力信息的机制与建模方法。枕下 BCG 信号并不是单一呼吸源的直接观测，而是呼吸运动、心冲击、头颈接触状态、机械传递路径和噪声共同作用后的混叠信号。传统低通滤波、EWT/IEWT 等方法依赖固定频带或频谱边界，在呼吸幅值较低、频率漂移、心冲击调制明显或信号形态非平稳时，容易出现频谱错分、双峰伪影和幅值趋势失真。

因此，本文的学术问题不是构建一个覆盖所有异常情况的工业级鲁棒系统，而是研究：

> 在 PSG 胸带弱同步参考条件下，如何从单通道枕下压电 BCG 中学习连续的呼吸努力估计信号；在胸带参考全流程跑通后，再将腹带参考和综合呼吸努力构造作为独立对比实验展开。

研究主线应保持连贯：

```text
BCG 混叠观测
  -> PSG 胸带弱同步监督
  -> 胸带参考下的呼吸努力估计信号
  -> 呼吸节律、努力包络和频谱结构评价
  -> 腹带参考与综合呼吸努力构造的独立对比实验
```

体动剔除、信号质量、心冲击可用性和低可信片段等问题需要被诚实说明，但它们不构成本文的研究主体。它们更适合作为实验对比、敏感性分析或讨论章节中的边界条件，用于解释模型在哪些条件下表现更好或更差，而不是把研究扩展成“BCG 可靠时如何做、不可靠时如何兜底”的工程化系统设计。

## 1. 研究定位

本课题面向枕下压电传感采集的 BCG 混叠信号，目标是从其中提取呼吸努力估计信号，用于反映受试者的呼吸节律和呼吸努力变化。

在当前数据条件下，任务不宜定义为“从 BCG 精确还原 PSG 胸带/腹带原始波形”，第一阶段更适合定义为：

> 基于弱同步 PSG 参考的枕下压电 BCG 呼吸努力估计信号建模。

模型需要输出的是一个在 BCG 时间轴上的连续呼吸努力估计信号，并可同时估计呼吸率和努力包络等辅助结果。第一阶段以 PSG 胸带为主要参考，使输出在窗口级、趋势级和频谱级与胸带参考一致，而不是追求逐点相位完全一致。腹带参考和综合呼吸努力参考不作为第一阶段模型的中间输出，而作为后续独立实验用于比较参考选择对结果的影响。

## 2. 数据前提与核心约束

### 2.1 PSG 与 BCG 未精确同步

两路信号未能获得精确同步，因此不应直接使用逐点波形监督作为主要训练目标。

不推荐的目标：

```text
BCG(t) -> 胸带(t)
BCG(t) -> 腹带(t)
BCG(t) -> RIPsum(t)
```

不推荐的主损失：

```text
MSE(预测呼吸波形, PSG 胸/腹带波形)
```

即使模型预测出了正确的呼吸节律，只要 BCG 与 PSG 存在秒级错位，逐点 MSE 也会错误惩罚模型，导致模型学习出过度平滑、幅值被压缩的平均波形。

### 2.2 体动已在信号处理初期剔除

体动剔除是本文的数据预处理前提，不作为本文的核心研究对象。需要保留以下信息，以避免弱同步监督和整夜评价被破坏：

- 原始时间戳；
- clean mask；
- 被剔除片段的起止位置；
- clean segment 与原始记录的映射关系；
- clean segment 边界附近是否存在体动剔除。

体动段的处理原则是保持原始时间关系清晰，而不是围绕其设计复杂兜底流程。若体动段已经被删除，应维护片段映射表；若体动段保留，则在训练和评价中使用 clean mask。体动剔除比例、边界影响和片段可用性可作为对比分析或讨论内容，不作为主模型设计目标。

### 2.3 单通道枕下信号的可辨识性有限

单通道枕下 BCG 不一定能唯一还原胸带和腹带两路独立运动。因此，当前阶段不宜把胸带估计、腹带估计和综合呼吸努力估计同时塞进同一个模型，并解释为模型完成了胸腹分离。

更稳妥的研究顺序是阶段化推进：

```text
阶段 A：以 PSG 胸带为唯一主要参考，完整跑通 BCG -> 呼吸努力估计信号的建模、训练和评价流程；
阶段 B：在相同流程下改用 PSG 腹带参考，作为独立对比实验；
阶段 C：在阶段 A/B 稳定后，再研究胸带、腹带如何构造综合呼吸努力参考，并训练或评价综合呼吸努力估计信号。
```

因此，`r_tho_hat(t)` 和 `r_abd_hat(t)` 不应被写成同一模型里的中间输出。它们分别代表不同参考设定下的独立实验输出。综合呼吸努力估计信号也应放在后续实验中讨论，而不是作为第一阶段必须完成的主输出。

## 3. 研究目标分层

### Level 1：呼吸节律估计

目标：

- 呼吸率轨迹；
- 呼吸周期长度；
- 呼吸主频变化；
- 窗口级呼吸次数。

该层最稳定，适合作为基础目标。

### Level 2：呼吸努力幅值趋势估计

目标：

- 呼吸努力强弱变化；
- 呼吸努力幅值减弱；
- 呼吸努力恢复性增强；
- 周期性呼吸或努力波动；
- 局部努力包络变化。

该层是课题的核心，但应采用相位不敏感监督，重点比较趋势、相对幅值和局部变化。

### Level 3：参考选择与综合呼吸努力构造对比

目标：

- 第一阶段先完成胸带参考下的呼吸努力估计信号；
- 第二阶段在同一流程下完成腹带参考实验；
- 第三阶段比较胸带参考、腹带参考与综合呼吸努力参考之间的关系；
- 研究归一化相加、质量加权、包络融合和学习式融合等综合构造路径。

该层是后续对比实验的一部分，但不应被表述为“单通道 BCG 必然可以精确分离胸腹两路”。胸带估计和腹带估计分别是不同参考设定下的实验结果，而不是同一模型内部的中间表示。

## 4. 信号信息利用原则

### 4.1 BCG 是耦合观测，不是简单加性噪声

枕下压电 BCG 可近似理解为以下成分的混合：

```text
呼吸运动 + 心冲击图 + 体动残余 + 接触状态变化 + 机械传递噪声 + 环境噪声
```

这些成分不是完全独立的。呼吸会调制心冲击幅值、基线、形态和能量分布；体位、枕头接触、头颈位置也会改变传递路径。因此，前处理阶段不宜强行做硬分离。

推荐做法是模型内部软分解：

```text
raw BCG
  -> respiratory latent branch
  -> optional high-frequency context branch
  -> time-frequency branch
  -> optional context branch
  -> gated fusion
  -> respiratory effort estimation signal / RR / envelope
```

### 4.2 呼吸分量

呼吸分量是主输出，但应定义为 BCG 时间轴上的呼吸努力估计信号，而不是 PSG 胸带或腹带的复制品。

主输出收敛到单一连续信号，其余量作为对它的可微读出，而非互相独立的预测头。理由是 A、RR、peak 在信息上都是 `r_hat(t)` 的派生量；分立多头只增加参数和"头间不一致"风险，弱同步的真正约束在于"监督挂在哪个相位不敏感量上"，而非"产出多少路独立信号"。

输出设计：

- `r_hat(t)`：唯一主输出，当前参考设定下的呼吸努力估计信号；
- `A_hat(t)`：呼吸努力包络，由 `r_hat(t)` 的包络（Hilbert / 滑窗 RMS）可微读出，不单独建头，避免包络与信号自相矛盾；
- `RR_hat(t)`：呼吸率轨迹，保留为一个轻量辅助头（给编码器稳健、易学的窗口级监督，起正则作用），或改为对 `r_hat(t)` 的瞬时频率读出，二选一；
- `P_breath(t)`：呼吸峰/谷或周期概率，从受监督主输出降级。弱同步下缺乏可靠峰/谷标签，单独监督会引入伪标签噪声；需要时作为后处理读出，不作为训练目标。

不同实验中的输出命名可区分为：

- `r_tho_hat(t)`：以 PSG 胸带为参考训练和评价的呼吸努力估计信号；
- `r_abd_hat(t)`：以 PSG 腹带为参考训练和评价的呼吸努力估计信号；
- `r_effort_hat(t)`：以综合呼吸努力参考训练和评价的呼吸努力估计信号。

上述三类输出对应三个独立实验设定，不建议在第一阶段同时作为多头输出。第一阶段默认 `r_hat(t) = r_tho_hat(t)`。

### 4.3 心冲击分量

心冲击不应简单视为噪声，但在当前数据集中也不应被设为必要输入或必要模块。若心冲击成分并不总是清晰可用，模型不应强依赖 J 峰、心搏位置或手工心冲击包络。

高频或心冲击候选频段的 CWT 时频图可以作为优先选择。CWT 幅值图已经包含：

- 高频能量随时间的变化；
- 各频率分量的幅度分布；
- 高频能量是否存在周期性；
- 高频能量是否被呼吸节律调制；
- 高频噪声、接触变化或残余运动的宽带能量特征。

心冲击包络本质上可以看作对心冲击候选频段时频能量的频率方向聚合：

```text
cardiac_envelope(t) = aggregate(CWT(t, f)), f in cardiac band
```

因此，当模型已经输入高频 CWT 幅值图时，额外输入心冲击包络可能是冗余的，也可能在心冲击不可用片段引入错误先验。

推荐定位：

- 必选：原始 BCG、呼吸候选频带和时频图；
- 可选：数据状态或上下文特征；
- 可选：高频/心冲击候选频段 CWT；
- 消融项：心冲击频带包络、J 峰、HR、心搏位置；
- 仅在 ECG 或心搏参考可靠时，才考虑心冲击辅助监督。

### 4.4 时频信息

时频图有价值，但不应作为唯一输入，也不应以“修复时频图再反变换”为主路线。

更合理的使用方式：

- 辅助识别呼吸频谱 ridge；
- 辅助观察呼吸频带能量连续性；
- 辅助区分呼吸、心冲击和残余噪声；
- 用窗口级频谱结构监督模型。

时频特征应与时域原始信号和呼吸候选频带共同输入模型。高频 CWT、数据状态特征可作为辅助上下文；心冲击包络不作为默认必选输入。

### 4.5 接触状态与低频漂移

低频漂移不一定全部是噪声，也可能反映：

- 头颈位置变化；
- 枕头受力变化；
- 体位变化；
- 传感器接触状态变化；
- 机械传递路径变化。

这类信息不适合作为呼吸主输出，但适合作为上下文特征，用于解释片段间差异。

## 5. PSG 参考标签构造

由于同步不精确，应优先构造相位不敏感标签。第一阶段以 PSG 胸带标签为主，腹带标签和综合呼吸努力标签留作后续独立对比实验。

### 5.1 参考构造顺序

第一阶段只使用 PSG 胸带构造主要参考标签，目标是先完整验证 BCG 到呼吸努力估计信号的全流程是否成立。此时不同时引入腹带分支或综合呼吸努力分支，避免模型目标过早复杂化。

建议实验顺序：

```text
A. 胸带参考实验:
   BCG -> r_tho_hat(t)

B. 腹带参考实验:
   BCG -> r_abd_hat(t)

C. 综合呼吸努力参考实验:
   BCG -> r_effort_hat(t)
```

综合呼吸努力参考不应简单固定为胸带和腹带的加性结果。胸带与腹带可能存在幅值差异、质量差异和局部相位差，直接相加有时会造成抵消或错误增强。

可比较以下构造方式：

```text
1. 归一化相加:
   RIPsum = normalized(胸带) + normalized(腹带)

2. 质量加权融合:
   R_effort = w_tho * normalized(胸带) + w_abd * normalized(腹带)

3. 包络融合:
   A_effort = combine(A_tho, A_abd)

4. 呼吸率/频谱层面鲁棒融合:
   RR_ref / Ridge_ref = robust_combine(胸带特征, 腹带特征)
```

其中归一化相加可作为基线，但不应被默认视为唯一正确的综合呼吸努力参考。当前更推荐先分别完成胸带参考和腹带参考实验，再比较不同融合策略对综合呼吸努力估计信号的影响。

注意不要只做全局归一化后丢弃幅值变化。呼吸努力减弱、恢复性增强等目标需要保留相对幅值趋势。

### 5.2 推荐标签

这里区分两个概念：**标签库**（离线全量提取，成本低，供评价与消融使用）与**训练目标**（实际进入损失函数的最小独立子集）。两者不要混为一谈：全部提取不等于全部监督。

第一阶段从 PSG 胸带离线提取以下标签，全部存入标签库备用：

- `RR_tho[k]`：胸带窗口级呼吸率；
- `A_tho[k]`：胸带努力幅值；
- `Drop_tho[k]`：胸带相对幅值下降比例；
- `Spec_tho[k, f]`：胸带呼吸频带归一化谱；
- `Ridge_tho[k]`：胸带主呼吸频率；
- `Recovery_tho[k]`：胸带恢复呼吸增强标签。

但其中存在硬冗余，不应全部作为训练目标：

- `RR` 与 `Ridge` 本质同一物理量（呼吸率 = 主频 × 60），仅单位/计算域不同，作两路损失是重复计数；
- `Drop`、`Recovery` 是包络 `A` 的派生视图（相对下降 / 下降后回升），非独立目标；
- `Spec`（频带归一化谱）已包含 `Ridge`（谱峰位置）。

因此第一阶段实际训练目标收敛为最小独立子集：

```text
训练目标 ≈ {节律: RR 或 Ridge 取一}
         + {努力: A 或其相对量 Drop}
         + {结构: Spec, 可选}
```

由于本数据集努力幅值只有相对/标定意义、无绝对单位，努力目标应作用在归一化/相对包络上（保留相对趋势，去除绝对尺度），与 5.1 末尾"不要丢弃幅值变化"的要求一致。其余标签（`Recovery`、`Drop` 检出能力、`Spec` 分布等）保留在标签库中作为评价指标与消融项，不进训练损失。

腹带实验与综合呼吸努力实验在各自独立实验中，按相同方式提取同名标签（`*_abd`、`*_effort`）并区分库与训练目标；综合呼吸努力标签可来自胸带/腹带的质量加权融合、鲁棒平均或学习式融合。三组标签分别对应独立实验，不作为第一阶段同一模型的多头监督。

这些相位不敏感标签比原始逐点波形更适合弱同步数据。

### 5.3 同步可用性门控（usable / unusable）

建议为每一晚或每个 clean segment 建立同步可用性标志。这里采用二元门控，而非三级分级。

原因：三级（S2/S1/S0）中 S2 与 S1 的区别只在"使用波形级（相位敏感）损失"时才成立。本数据集的 lag 不固定但基本在 3 秒以内，而呼吸周期约 3–5 秒，3 秒的 lag 在波形层面接近半个到一个周期（可能近反相），逐点/波形监督在此条件下不可靠；但在 30–60 秒窗口层面，3 秒仅是约 5–10% 的偏移，窗口级统计标签依然可信。因此"逐点同步可信（原 S2）"在本数据集中近乎空集，S2 与 S1 应合并。真正需要区分的只有"窗口级可用"与"完全不可用"。

#### usable：窗口级同步可用

特征：

- 已知 BCG 与 PSG 属于大致相同时间段；
- 最佳 lag 已估计且在合理上界（≤ 约 3 秒）内；
- 秒级峰谷不一定可靠，但窗口级统计（呼吸率、包络趋势、频谱）可信。

可用监督：

- 呼吸率损失；
- 包络趋势损失；
- 频谱 / ridge 结构损失。

不使用逐点波形损失（波形损失仅作为消融项，见 8.4）。

#### unusable：同步关系不可信

特征：

- 无法确认 BCG 与 PSG 的时间对应关系；
- 窗口级对应也不可靠（如审计中 lag 无稳定峰、相关性过低）。

可用方式：

- BCG 自监督 / 表征学习（见扩展章节）；
- record-level 或整夜呼吸节律统计标签。

不应声称从 unusable 片段学习到了逐时呼吸努力变化。

> 说明：原三级体系的中间档（S2 逐点可信）与 8.4 波形损失共存亡——若波形损失仅留作消融、不进主方法，则 S2/S1 的区分失去意义。同步审计仍需估计每段最佳 lag（见 Step 1），但其产出简化为 `{lag, usable 标志}` 二元门控，而非三级标签。

## 6. 推荐模型：WeakSync-RespBCG-Net

### 6.1 输入

```text
x_raw       : 原始 BCG clean 片段
x_resp      : BCG 呼吸候选频带
x_hf_cwt    : 高频/心冲击候选频段 CWT 时频图（可选）
TF          : 全频或呼吸相关 STFT / CWT / SST 时频图
context_pre : rolling energy / STD / derivative energy / clean mask（可选，用于补充分析）
```

其中 `context_pre` 主要用于描述数据状态和进行对比分析，不应把模型主线扩展为复杂的质量兜底系统。

### 6.2 编码器

- 1D Temporal Encoder：学习时域节律、局部形态和长程变化；
- 2D Time-Frequency Encoder：学习呼吸 ridge、频带能量和非平稳变化；
- High-Frequency Context Encoder：可选，用于学习高频/心冲击候选频段的时频上下文；
- Quality/Context Encoder：可选，用于分析接触状态、低频漂移和片段差异对结果的影响。

### 6.3 融合方式

推荐使用 gated fusion 或 cross-attention。融合模块的核心作用是整合时域、时频域和可选上下文信息，而不是构建复杂的质量兜底策略，也不是在第一阶段同时学习胸带/腹带多路输出。

例如：

```text
raw/time branch + TF branch + optional high-frequency TF branch + optional context branch
                    -> gated fusion
                    -> multi-task heads
```

不同参考实验共享同一套主体框架，但分别训练和评价：

```text
胸带参考实验: BCG -> r_tho_hat(t), A_tho_hat(t), RR_tho_hat(t)
腹带参考实验: BCG -> r_abd_hat(t), A_abd_hat(t), RR_abd_hat(t)
综合呼吸努力实验: BCG -> r_effort_hat(t), A_effort_hat(t), RR_effort_hat(t)
```

综合呼吸努力实验可比较多种构造策略，包括归一化相加、质量加权、包络融合和学习式融合。该比较本身可以作为方法研究的一部分，但应放在胸带参考流程跑通之后。

### 6.4 输出

主输出单一，其余量为对它的可微读出或轻量辅助头（与 4.2 一致）：

```text
r_hat(t)     : 唯一主输出，BCG 时间轴上的呼吸努力估计信号
A_hat(t)     : 呼吸努力包络，由 r_hat(t) 包络读出，不单独建头
RR_hat(t)    : 呼吸率轨迹，轻量辅助头或瞬时频率读出（二选一）
P_breath(t)  : 呼吸峰/谷或周期概率，后处理读出，不作受监督主输出
```

不同参考实验中的主输出：

```text
r_tho_hat(t)    : 胸带参考实验的呼吸努力估计信号
r_abd_hat(t)    : 腹带参考实验的呼吸努力估计信号
r_effort_hat(t) : 综合呼吸努力参考实验的呼吸努力估计信号
```

第一阶段最终输出以 `r_tho_hat(t)` 为主。`r_abd_hat(t)` 和 `r_effort_hat(t)` 是后续独立实验输出，不作为第一阶段模型的中间输出或辅助头。

可选输出：

```text
Q_hat(t)          : 质量或不确定性指标（用于对比分析）
HF_context(t)     : 高频上下文指标（用于对比分析）
Card_env_hat(t)   : 心冲击包络（仅作为消融项）
HR_hat(t)         : 心率轨迹（仅在 ECG 或心搏参考可靠时）
J_peak_hat(t)     : 心搏位置概率（仅在 ECG 或心搏参考可靠时）
```

## 7. 窗口设计

推荐采用多尺度窗口。

### 短窗：8-16 秒

用于：

- 局部波形；
- 高频局部结构；
- 呼吸峰谷；
- 短时片段特征分析。

### 中窗：30-60 秒

用于：

- 呼吸率估计；
- 呼吸频谱 ridge；
- 包络趋势。

### 长窗：90-180 秒

用于：

- 呼吸努力持续减弱趋势；
- 呼吸努力恢复性增强；
- 周期性呼吸；
- 长程努力变化。

训练时不建议只随机切短窗，否则容易丢失长程趋势和幅值变化。

## 8. 损失函数设计

损失设计遵循"核心最小、其余消融"的原则，避免堆叠互相冗余或与弱同步主线无关的项。主损失收敛为核心 2–3 项加一个小权重正则项：

```text
L =
  λ_env  * L_env      # 努力包络/相对幅值趋势——论文头号主张，作用在归一化/相对包络上
+ λ_rr   * L_RR       # 窗口级呼吸率（节律）
+ λ_sm   * L_smooth   # 小权重平滑正则
( + λ_spec * L_spec ) # 可选结构约束，与 L_RR 部分重叠
```

裁定说明：

- `L_env` 升为核心。它直接监督"呼吸努力"这一标题量，且包络对秒级 lag（≤ 约 3 秒）稳健。本数据集努力幅值只有相对意义，故作用在归一化/相对包络上。
- `L_RR` 保留，提供稳健的节律监督。
- `L_spec` 降为可选；与 `L_RR` 在主频信息上重叠，保留时仅作结构约束。
- `L_ridge` 删除，被 `L_spec` / `L_RR` 覆盖（见 8.3）。
- `L_smooth` 保留但须小权重，只压制高频抖动噪声，不得压制幅值趋势，否则会重现本文反对的"过度平滑"失败模式。
- `L_shift_wave` 不进主损失，仅作为消融项（见 8.4）。
- 心冲击相关辅助损失、不确定性损失不作为默认项，只作为消融或扩展分析加入。

### 8.1 呼吸率损失

```text
L_RR = MAE(RR_hat, RR_ref)
```

若存在局部错位，可在受限窗口内使用允许偏移的损失：

```text
L_RR = min over τ MAE(RR_hat(t), RR_ref(t + τ))
```

也可在 RR 轨迹上使用 Soft-DTW，但需要限制 warping window，避免过度扭曲。

### 8.2 包络趋势损失

```text
L_env = 1 - Corr(A_hat, A_ref)
```

或使用相对幅值下降：

```text
Drop = 当前努力幅值 / 局部基线努力幅值
L_drop = Huber(Drop_hat, Drop_ref)
```

该损失适合努力下降和恢复呼吸增强任务。

### 8.3 频谱结构损失（可选）

可比较呼吸频带内的归一化频谱：

```text
L_spec = KL(P_hat(f | window), P_ref(f | window))
```

频谱损失只作为可选结构约束，不能替代包络和呼吸率监督。

注：原"主呼吸频率损失" `L_ridge = MAE(f_resp_hat, f_resp_ref)` 已删除——主频信息与 `L_RR`（呼吸率 = 主频 × 60）、`L_spec`（谱峰位置）重叠，单列会造成对同一物理量的重复监督。

### 8.4 Shift-invariant waveform loss（仅消融项，不进主方法）

该损失不进入主损失，仅作为消融实验保留。形式为：

```text
L_shift = min over τ [1 - Corr(r_hat(t), y_ref(t + τ))]
```

移出主方法的理由：

- 它重新引入了本文明确反对的逐点波形匹配；
- 本数据集 lag 不固定但约在 3 秒内，而呼吸周期约 3–5 秒，3 秒 lag 在波形层面接近半个到一个周期（可能近反相），即便允许单一全局 τ 对齐也无法纠正窗口内的 drift / warping；
- 它依赖"逐点同步可信"片段，而该类片段在本数据集中近乎空集（见 5.3）。

消融定位：作为"在窗口级同步可用片段上额外加波形损失是否带来增益"的对照。预期结论很可能是"无增益或反而变差"——若如此，恰好正向支持本文"应使用相位不敏感监督"的核心主张，这是保留它作消融的唯一科研价值。

注：若同步审计（Step 1）显示窗口级可用数据中几乎不存在逐点可信片段，则本消融可直接省略，不必强行构造。

### 8.5 不确定性分析项

若需要讨论信号质量对结果的影响，可额外输出 `Q_hat(t)` 或不确定性 `s(t)`，并作为对比分析项。可采用：

```text
L = exp(-s) * L_task + s
```

该形式可以分析低质量片段对估计结果的影响，但不应成为本文主模型的必要组成部分。

### 8.6 高频/心冲击辅助项

高频/心冲击信息优先通过 CWT 时频图输入，不默认要求额外构造心冲击包络、J 峰或 HR 标签。

建议做消融比较：

```text
A: raw BCG + respiratory band
B: A + full / high-frequency CWT
C: B + cardiac-band pooled envelope
D: B + ECG/心搏辅助监督（仅在参考可靠时）
```

若有 ECG 或可靠心搏参考，可作为扩展项加入：

- HR 损失；
- J 峰检测损失；
- 心搏间期损失。

若没有可靠心搏参考，不建议强行引入心冲击辅助监督。心冲击包络最多作为可选派生特征或消融实验项；只有当实验表明它能稳定提升呼吸率、努力包络或数据状态分层表现时，再保留。

### 8.7 参考通道切换实验

胸带、腹带和综合呼吸努力不应在第一阶段写成同一模型的辅助损失。更合适的做法是保持模型主体、输入和评价指标基本一致，只切换参考标签，形成独立对比实验。

```text
实验 A:
  L_tho = L_weak(r_tho_hat, thoracic_ref)

实验 B:
  L_abd = L_weak(r_abd_hat, abdominal_ref)

实验 C:
  L_effort = L_weak(r_effort_hat, effort_ref)
```

这里的 `L_weak` 不应默认采用逐点 MSE，也不默认包含波形损失，而应优先使用包络趋势、呼吸率和可选的频谱结构监督（即 8 节开头的核心损失组合）。波形损失仅在消融中考察（见 8.4）。三组实验的区别在于参考来源不同，而不是同一模型内部有胸带、腹带和综合呼吸努力三个中间输出。

当前优先完成实验 A。实验 B 和实验 C 在实验 A 全流程稳定后再开启，用于回答“参考选择是否改变 BCG 呼吸努力估计结果”以及“综合呼吸努力参考如何构造更合理”。

## 9. 扩展与未来工作

本章收纳不属于核心主线、但可作为消融或后续探索的内容。它们都不进入第一阶段主方法、主干网络或默认损失函数；仅在核心闭环（BCG → 胸带参考呼吸努力估计信号）跑通后，按需作为扩展实验展开。

### 9.1 对比学习预训练

定位：标注稀缺、同步不精确时的 encoder 增强手段，不支撑核心主张。仅当实测中弱同步标签不足、监督训练难以收敛时再回收。

对比学习适合在同步不精确、标注有限的条件下学习稳定表征。

可设计正样本：

- 同一 BCG 片段的不同增强版本；
- 同一粗时间窗口内的 BCG 与 PSG 特征；
- 同一受试者同一睡眠阶段的相邻片段；
- 呼吸率或努力包络相近的片段。

可设计增强：

- 幅值缩放；
- 时间裁剪；
- 轻微时间扭曲；
- 噪声注入；
- 频带遮蔽；
- 时频图遮蔽。

对比学习用于 encoder 预训练，最终仍需要 PSG 弱同步标签进行监督微调。

### 9.2 其他扩展项汇总

以下内容在正文相应位置已说明定位，集中列在此处以明确它们均属扩展，不进主方法：

- 波形损失消融（见 8.4）：在窗口级同步可用片段上额外加 shift-invariant waveform loss，考察是否带来增益；若同步审计显示几乎无逐点可信片段，可直接省略。
- 高频 / 心冲击辅助监督（见 8.6）：HR、J 峰、心搏间期损失，仅在有可靠 ECG / 心搏参考时作为消融项。
- 不确定性 / 质量分析项（见 8.5）：输出 `Q_hat(t)` 或不确定性 `s(t)`，用于分析低质量片段对结果的影响，非主模型必要组成。
- GAN 生成约束（见 13.4）：作为探索性猜想，仅在扩展或消融中以小权重弱正则出现，且判别器须 conditional on BCG / 节律 / 上下文，避免只奖励"平滑好看"的波形。

## 10. 当前研究边界

当前研究只关注呼吸努力估计信号提取，不把其他下游任务作为模型输出、训练目标或主评价内容。

因此，当前模型只围绕信号估计本身展开，重点输出：

- 呼吸努力估计信号；
- 呼吸率；
- 呼吸努力包络；
- 呼吸峰/谷或周期概率。

腹带参考和综合呼吸努力参考作为后续独立对比实验展开，不作为第一阶段模型的中间输出或辅助监督。

其他下游应用可作为后续独立课题或应用扩展，但不纳入本文档定义的当前研究任务。当前任务的核心是证明：BCG 中可以提取出稳定、连续、与 PSG 胸带弱同步参考一致的呼吸努力估计信号。

## 11. 评价指标

### 11.1 呼吸节律指标

- RR MAE；
- RR RMSE；
- breath count error；
- dominant respiratory frequency error。

建议按 30 秒或 60 秒窗口评价。

### 11.2 努力幅值指标

- 包络趋势相关性；
- 幅值下降检出能力；
- 恢复性增强检出能力；
- 局部相对幅值误差；
- 周期性呼吸趋势一致性。

### 11.3 频谱结构指标

- 呼吸主频误差；
- 呼吸频带能量分布一致性；
- 时频 ridge 连续性；
- 多分辨率频谱相似性。

### 11.4 数据覆盖与敏感性分析

由于体动已剔除，可在补充实验或讨论中报告：

- 有效分析时长占比；
- 被剔除片段占比；
- 每夜可用数据比例；
- 不同数据状态分层下的性能。

这些指标用于解释实验样本边界和结果适用范围，不作为本文主模型的核心评价目标。

### 11.5 波形指标

波形相关性只能作为辅助指标，并且需要满足：

- 只在窗口级同步可用（usable）片段上、且允许最佳 lag 对齐后计算；
- 仅用于波形损失消融（见 8.4）的对照评估；
- 不作为主要结论依据。

## 12. 实施路线

### Step 1：同步审计（lag 估计 + 二元门控）

对每晚和每个 clean segment：

- 提取 BCG 低频候选呼吸信号；
- 第一阶段构造 PSG 胸带参考；
- 后续实验再构造 PSG 腹带参考和候选综合呼吸努力参考；
- 计算 cross-correlation；
- 估计最佳 lag（预期上界约 3 秒）；
- 统计最大相关系数，判定该段是否 usable。

该审计是承重环节：同步未知时必须先粗对齐，才能知道哪个 PSG 窗口对应哪个 BCG 窗口，否则窗口级标签无法构造。但其产出简化为二元门控，不再做三级分类：

- 每段最佳 lag（用于设定窗口级标签的 ±lag 对齐范围）；
- 每段 usable / unusable 标志（见 5.3）；
- 数据级可用比例报告（usable 占比、lag 分布、是否存在固定 offset 或缓慢 drift）。

### Step 2：构造弱同步标签

第一阶段从 PSG 胸带提取标签存入标签库，但训练只用最小独立子集（见 5.2）：

- 训练目标：`RR_tho`（或 `Ridge_tho` 取一）+ `A_tho`（归一化/相对包络）+ 可选 `Spec_tho`；
- 仅入库备用（评价/消融）：`Drop_tho`、`Recovery_tho`、`Ridge_tho`（若以 RR 为训练目标）等。

腹带标签和综合呼吸努力标签只在后续独立实验中构造。不要先训练逐点波形回归，也不要在第一阶段把胸带、腹带和综合呼吸努力同时作为多头监督。

### Step 3：平凡基线冒烟测试

在投入深度模型前，先用一个平凡基线验证评价管线与标签质量：

- 低通滤波 + Hilbert 估计 RR / 包络。

若连简单方法都得不到合理 RR，说明标签或指标存在 bug，应在此处先暴露，而非投入深度模型后才发现。完整传统基线套件（EWT / IEWT / CWT-SST ridge tracking 等）后移到 Step 6 之后，作为最终对比材料。

### Step 4：训练弱同步多任务模型

第一阶段模型以胸带参考训练，主输出与辅助量（见 4.2 / 6.4）：

- `r_tho_hat(t)`：唯一主输出；
- `A_tho_hat(t)`：由 `r_tho_hat` 包络读出；
- `RR_tho_hat`：轻量辅助头或瞬时频率读出；
- optional context indicators。

此阶段不强求 `r_hat(t)` 与 PSG 逐点对齐。

同时应进行输入消融，判断高频信息是否真的带来增益：

- `raw BCG + respiratory band`；
- 上述输入 + 全频或高频 CWT；
- 上述输入 + 心冲击频带聚合包络；
- 上述输入 + ECG/心搏辅助监督，仅在参考可靠时使用。

若心冲击包络或心搏辅助监督没有稳定提升呼吸率、努力包络、频谱结构或数据状态分层表现，则不应纳入默认模型。

### Step 5：跑通胸带参考完整流程

第一阶段只围绕胸带参考完成完整流程：

```text
BCG 输入
  -> 弱同步多任务模型
  -> r_tho_hat(t) / A_tho_hat(t) / RR_tho_hat(t)
  -> 片段级评价
  -> 整夜连续评价
```

这一阶段的目标是验证 BCG 到胸带参考呼吸努力估计信号的建模闭环，包括标签构造、同步审计（二元门控）、训练、平凡基线对比和整夜输出评价。

### Step 6：开启腹带与综合呼吸努力独立实验

胸带参考流程稳定后，再开启两个独立实验：

```text
实验 B：BCG -> r_abd_hat(t)
实验 C：BCG -> r_effort_hat(t)
```

实验 B 使用腹带参考重复同一套训练和评价流程，用于比较胸带参考与腹带参考的差异。实验 C 研究综合呼吸努力参考的构造方式，需要比较以下策略：

- 归一化相加；
- 质量加权融合；
- 包络融合；
- 学习式融合模块。

综合呼吸努力实验可使用以下约束（与第一阶段一致的相位不敏感监督）：

- envelope consistency（包络一致性）；
- spectral consistency（频谱一致性）；
- smoothness prior（小权重平滑正则）；
- breath peak weak labels（可选弱标签）。

shift-invariant waveform loss 不进默认监督，仅作消融（见 8.4）。不要使用全数据逐点 MSE 作为主监督。

### Step 7：完整传统基线对比

在模型流程跑通、评价管线稳定后，补齐完整传统基线套件作为最终对比材料（仅作对照，非证明更优）：

- 低通滤波；
- EWT；
- IEWT；
- CWT / SST ridge tracking；
- Hilbert-based RR；
- 简单包络估计。

传统方法同样使用弱同步评价指标（窗口级、允许 lag 对齐），避免用未对齐波形相关性直接否定其表现。

### Step 8：整夜连续评估

在完成片段级建模后，应评估模型在整夜数据上的连续输出质量：

- 第一阶段：`r_tho_hat(t)`；
- 后续实验：`r_abd_hat(t)`、`r_effort_hat(t)`；
- `A_hat(t)`；
- `RR_hat(t)`；
- context / clean mask。

最终评价呼吸节律、努力包络、频谱结构，波形一致性仅作辅助指标。有效覆盖率和数据状态分层可作为补充分析。跨实验比较时，应明确不同参考来源造成的差异，而不是把腹带或综合呼吸努力结果解释为胸带模型的中间结果。

## 13. 关键假设与对比讨论

### 13.1 未精确同步导致错误监督

讨论点：

- 逐点 MSE 将同步误差误判为模型误差；
- 模型输出被迫平滑化；
- 呼吸峰谷和幅值下降被削弱。

论文处理：

- 建立同步审计与二元可用性门控（usable / unusable）；
- 主用窗口级、频谱级和包络级监督；
- 波形损失不进主方法，仅作消融。

### 13.2 单通道 BCG 无法唯一还原胸腹两路

讨论点：

- 模型根据人群先验猜测胸/腹分量；
- 胸腹反常运动无法可靠恢复。

论文处理：

- 第一阶段以胸带参考呼吸努力估计信号为主；
- 腹带参考和综合呼吸努力参考作为后续独立对比实验；
- 不把胸带/腹带估计写成同一模型内部的分离结果；
- 不确定性分析作为补充讨论，而不是主模型目标。

### 13.3 时频图修复可能丢失相位和幅值意义

讨论点：

- 频谱看似合理，但时域幅值和波形变化不可信；
- 反变换后的时域波形缺乏生理解释。

论文处理：

- 时频图作为辅助输入；
- 输出仍以 RR、包络和呼吸努力估计信号为主；
- 频谱损失不单独作为主目标。

### 13.4 GAN 生成约束作为探索性猜想

讨论点：

- GAN 可以被视为一种探索性猜想：用生成约束帮助模型输出更接近真实呼吸努力形态的连续信号；
- 该猜想尚不构成本文主方法，也不应放在模型主干或默认损失函数中；
- 生成正常呼吸样式；
- 掩盖努力减弱、恢复性增强或局部幅值波动；
- 使呼吸努力估计结果偏乐观。

论文处理：

- GAN 放在扩展实验、消融实验或讨论章节中；
- 若使用，只作为小权重弱正则或对比方法；
- 判别器应 conditional on BCG、呼吸节律或上下文特征，避免只奖励“平滑好看”的波形；
- 主监督保持在呼吸率、努力包络和频谱结构目标；
- 必须用信号估计指标验证，而不能只看波形是否平滑。

### 13.5 体动剔除导致评价偏差

讨论点：

- 只在容易片段上评价；
- 大量剔除导致整夜可用性被高估。

论文处理：

- 在补充实验中报告有效覆盖率；
- 在讨论中分析数据状态分层对结果的影响。

## 14. 最终建议

本课题的核心主张应从“恢复一个漂亮的呼吸波形”转为：

> 模型能够从枕下压电 BCG 中提取胸带参考下的呼吸努力估计信号，稳定估计呼吸节律、努力幅值变化和频谱结构；在体动剔除后的有效片段中，这些输出在窗口级、趋势级和频谱级与 PSG 胸带参考一致，并优于低通、EWT、IEWT 等传统方法。

推荐主路线：

```text
BCG 多视图输入
  -> 弱同步多任务网络
  -> 胸带参考呼吸努力估计信号 / RR / 努力包络
  -> 整夜连续呼吸努力估计
  -> 腹带参考与综合呼吸努力参考的独立对比实验
```

推荐模型重点：

- 不以逐点波形还原为核心；
- 不把时频图修复作为唯一目标；
- 不把心冲击包络、J 峰或 HR 设为必要依赖；
- 不把胸带/腹带估计写成同一模型的中间输出或确定性分离结果；
- 不把体动或低质量片段的兜底处理作为研究主体；
- 优先学习相位不敏感的呼吸节律、幅值趋势和频谱结构。

这一路线更符合当前数据条件，也更容易形成清晰、可信、可评价的研究结果。

## 15. 方案评审结论（精简与重排）

本节针对第 4、5、8、9、12 节的复杂度做评审，原则是"先证明核心主张、再扩展"，删除与弱同步主线无关或互相冗余的设计。本节为评审决策记录；相应裁定已落地到正文对应各节（4.2 / 5.2 / 5.3 / 6.4 / 第 8 章 / 第 9 章 / 11.5 / 第 12 章 / 13.1），本节保留作为变更依据与理由说明。

### 15.1 输出精简（对应 4.2）

结论：主输出收敛到 `r_hat(t)` 一个连续信号，其余量改为对它的可微读出，而非互相独立的预测头。

- `r_hat(t)`：唯一主输出（呼吸努力估计信号）。
- `A_hat(t)`：由 `r_hat(t)` 的包络（Hilbert / RMS）可微读出，不单独建头，避免包络与信号自相矛盾。
- `RR_hat(t)`：保留为一个轻量辅助头（给编码器稳健、易学的窗口级监督，起正则作用），或改为对 `r_hat(t)` 的瞬时频率读出。二选一。
- `P_breath(t)`：从主输出降级。弱同步下缺乏可靠峰/谷标签，单独监督会引入伪标签噪声；需要时作为后处理读出，不作为受监督主输出。

理由：A / RR / peak 在信息上都是 `r_hat` 的派生量。分立多头只增加参数与"头间不一致"风险。弱同步的真正约束在于"监督挂在哪个相位不敏感量上"，而非"产出多少路独立信号"。

### 15.2 标签精简（对应 5.2）

结论：区分"标签库"与"训练目标"。六类标签可全部离线提取备用（成本低，供评价与消融），但训练监督只用最小独立子集。"先提取备用"的判断成立。

冗余关系：

- `RR` 与 `Ridge` 本质同一物理量（呼吸率 = 主频 × 60），仅单位/计算域不同，作两路损失是重复计数。
- `Drop`、`Recovery` 是包络 `A` 的派生视图（相对下降 / 下降后回升），非独立目标。
- `Spec`（频带归一化谱）已包含 `Ridge`（谱峰位置）。

独立训练目标 ≈ {节律：`RR` 或 `Ridge` 取一} + {努力：`A` 或其相对量 `Drop`} + {结构：`Spec`，可选}。其余作为评价指标与消融项保留在标签库，不进训练损失。

### 15.3 同步可信度分级（对应 5.3）

结论：三级（S2/S1/S0）过度设计，简化为二元门控。

- S2 与 S1 的区别只有在"使用波形级（相位敏感）损失"时才成立；若按本文主张采用相位不敏感（窗口级）监督，S2 与 S1 用同一套损失，二者合并。
- 真正需要的只是：该片段窗口级同步是否可信（可挂窗口级标签）/ 不可信（仅自监督或弃用），即 `usable` 与否。
- 因此建议：保留二元 `usable` 标志 + 每段最佳 lag，废弃三级标签体系。三级体系的中间复杂度（S2 vs S1）与 8.4 波形损失共存亡。

### 15.4 损失函数重构（对应第 8 节）

结论：损失从 5 项收敛到核心 2–3 项 + 1 正则项；8.4 移出主方法。

精简后主损失：

```text
L = λ_env  * L_env      # 努力包络/相对幅值趋势——论文头号主张，建议用归一化/相对包络以保留趋势
  + λ_rr   * L_RR       # 窗口级呼吸率（节律）
  + λ_sm   * L_smooth   # 小权重平滑正则（权重须小，否则重现"过度平滑"失败模式）
( + λ_spec * L_spec )   # 可选结构约束，与 L_RR 部分重叠
```

逐项裁定：

- `L_env`：保留并升为核心。直接监督"呼吸努力"这一标题量；包络对秒级 lag 稳健。建议作用在相对/归一化包络上，兼顾趋势保留与尺度鲁棒。
- `L_RR`：保留，节律监督，稳健。
- `L_spec`：降为可选。与 `L_RR` 重叠（峰位置）；保留则作结构约束。
- `L_ridge`：删除，被 `L_spec` / `L_RR` 覆盖。
- `L_smooth`：保留但小权重。注意它与"反对过度平滑"存在张力——应只压制高频抖动噪声，不得压制幅值趋势。
- `L_shift_wave`（8.4）：移出主方法。它重新引入了本文明确反对的逐点波形匹配，且依赖 S2 分级机制；单一全局 lag 无法纠正窗口内 drift / warping。**仅作为消融项**保留（"在高同步片段加波形损失是否有增益"）——若结论为"无增益/反而变差"，恰好正向支持本文主张，这是它唯一的科研价值。

### 15.5 对比学习预训练（对应第 9 节）

结论：移入扩展/未来工作。它是"标注稀缺时的编码器增强手段"，不支撑核心主张（BCG → 努力信号的弱同步建模）。仅当实测中标签数据稀缺导致模型无法训练时再回收。

### 15.6 同步可信度审计（对应 Step 1）

结论：审计保留（承重），但产出简化。

- lag 估计是承重环节：同步未知时，必须先粗对齐才能知道"哪个 PSG 窗口对应哪个 BCG 窗口"，否则窗口级标签根本无法构造。故 cross-correlation / 最佳 lag 估计不可删。
- 但产出从"S2/S1/S0 三级标签"简化为"{每段最佳 lag, usable 标志} + 数据级可用比例报告"。
- 即：保留"审计"，去掉"三级分类法"。

### 15.7 传统基线重排（对应 Step 3）

结论：完整基线套件后移，但保留一个平凡基线前置。

- 完整套件（EWT / IEWT / CWT-SST ridge / Hilbert-RR）后移到模型流程跑通之后，作为最终对比材料。
- 但保留一个平凡基线（低通 + Hilbert RR）前置，作为评价管线与标签质量的冒烟测试：若简单方法都得不到合理 RR，说明标签或指标有 bug，应在投入深度模型前先暴露。

### 15.8 重排后的实施路线（已落地于第 12 节）

```text
Step 1. 同步审计：估计每段 lag + usable 二元标志，报告可用比例
Step 2. 构造弱同步标签：最小训练子集（RR/Ridge 取一 + 归一化包络 + 可选 Spec），其余入标签库备用
Step 3. 平凡基线冒烟测试：低通 + Hilbert，校验评价管线与标签质量
Step 4. 训练弱同步多任务模型（胸带参考）+ 输入消融
Step 5. 跑通胸带参考完整流程（片段级 + 整夜）
Step 6. 腹带 / 综合呼吸努力独立实验（实验 B / C）
Step 7. 完整传统基线对比（EWT / IEWT / CWT-SST / Hilbert-RR，仅作对照）
Step 8. 整夜连续评估（覆盖全部实验输出）
扩展.   对比学习预训练、波形损失消融、GAN、不确定性（见第 9 章）
```

### 15.9 开放问题的确认结论

三项开放问题已确认，裁定相应锁定：

1. 同步误差性质：lag 不固定（有多有少）但基本在 3 秒以内。结论：窗口级（30–60s）标签可信，逐点波形监督不可靠；"逐点同步可信"片段近乎空集，5.3 二元门控成立，8.4 波形损失仅留消融、若审计显示无逐点可信片段可直接省略。
2. 努力幅值：只有相对/标定意义，无绝对单位。结论：`L_env` 作用在归一化/相对包络上（保留相对趋势、去除绝对尺度），见 8 节与 5.2。
3. 传统基线定位：不需证明"优于 EWT/IEWT"，仅作对比对象（本研究定性为试验）。结论：完整基线套件后移至 Step 7、降为参照物；仅保留低通+Hilbert 平凡基线前置当冒烟测试（Step 3）。

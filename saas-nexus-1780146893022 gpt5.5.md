# 枕下压电 BCG 呼吸信号提取研究方案

## 0. 论文研究背景与主线

本研究面向睡眠场景下的枕下压电 BCG 信号，关注其在非接触或弱接触条件下承载呼吸努力信息的机制与建模方法。枕下 BCG 信号并不是单一呼吸源的直接观测，而是呼吸运动、心冲击、头颈接触状态、机械传递路径和噪声共同作用后的混叠信号。传统低通滤波、EWT/IEWT 等方法依赖固定频带或频谱边界，在呼吸幅值较低、频率漂移、心冲击调制明显或信号形态非平稳时，容易出现频谱错分、双峰伪影和幅值趋势失真。

因此，本文的学术问题不是构建一个覆盖所有异常情况的工业级鲁棒系统，而是研究：

> 在 PSG 胸带弱同步参考条件下，如何从单通道枕下压电 BCG 中学习连续的呼吸努力估计信号；在胸带参考全流程跑通后，再将腹带参考和综合呼吸努力构造作为独立对比实验展开。

研究主线应保持连贯：

```text
BCG 混叠观测
  -> PSG 胸带弱同步监督
  -> 胸带参考下的呼吸努力估计信号
  -> 呼吸节律、努力包络和频谱结构评价
  -> 腹带参考与综合呼吸努力构造的独立对比实验
```

体动剔除、信号质量、心冲击可用性和低可信片段等问题需要被诚实说明，但它们不构成本文的研究主体。它们更适合作为实验对比、敏感性分析或讨论章节中的边界条件，用于解释模型在哪些条件下表现更好或更差，而不是把研究扩展成“BCG 可靠时如何做、不可靠时如何兜底”的工程化系统设计。

## 1. 研究定位

本课题面向枕下压电传感采集的 BCG 混叠信号，目标是从其中提取呼吸努力估计信号，用于反映受试者的呼吸节律和呼吸努力变化。

在当前数据条件下，任务不宜定义为“从 BCG 精确还原 PSG 胸带/腹带原始波形”，第一阶段更适合定义为：

> 基于弱同步 PSG 参考的枕下压电 BCG 呼吸努力估计信号建模。

模型需要输出的是一个在 BCG 时间轴上的连续呼吸努力估计信号，并可同时估计呼吸率和努力包络等辅助结果。第一阶段以 PSG 胸带为主要参考，使输出在窗口级、趋势级和频谱级与胸带参考一致，而不是追求逐点相位完全一致。腹带参考和综合呼吸努力参考不作为第一阶段模型的中间输出，而作为后续独立实验用于比较参考选择对结果的影响。

## 2. 数据前提与核心约束

### 2.1 PSG 与 BCG 未精确同步

两路信号未能获得精确同步，因此不应直接使用逐点波形监督作为主要训练目标。

不推荐的目标：

```text
BCG(t) -> 胸带(t)
BCG(t) -> 腹带(t)
BCG(t) -> RIPsum(t)
```

不推荐的主损失：

```text
MSE(预测呼吸波形, PSG 胸/腹带波形)
```

即使模型预测出了正确的呼吸节律，只要 BCG 与 PSG 存在秒级错位，逐点 MSE 也会错误惩罚模型，导致模型学习出过度平滑、幅值被压缩的平均波形。

### 2.2 体动已在信号处理初期剔除

体动剔除是本文的数据预处理前提，不作为本文的核心研究对象。需要保留以下信息，以避免弱同步监督和整夜评价被破坏：

- 原始时间戳；
- clean mask；
- 被剔除片段的起止位置；
- clean segment 与原始记录的映射关系；
- clean segment 边界附近是否存在体动剔除。

体动段的处理原则是保持原始时间关系清晰，而不是围绕其设计复杂兜底流程。若体动段已经被删除，应维护片段映射表；若体动段保留，则在训练和评价中使用 clean mask。体动剔除比例、边界影响和片段可用性可作为对比分析或讨论内容，不作为主模型设计目标。

### 2.3 单通道枕下信号的可辨识性有限

单通道枕下 BCG 不一定能唯一还原胸带和腹带两路独立运动。因此，当前阶段不宜把胸带估计、腹带估计和综合呼吸努力估计同时塞进同一个模型，并解释为模型完成了胸腹分离。

更稳妥的研究顺序是阶段化推进：

```text
阶段 A：以 PSG 胸带为唯一主要参考，完整跑通 BCG -> 呼吸努力估计信号的建模、训练和评价流程；
阶段 B：在相同流程下改用 PSG 腹带参考，作为独立对比实验；
阶段 C：在阶段 A/B 稳定后，再研究胸带、腹带如何构造综合呼吸努力参考，并训练或评价综合呼吸努力估计信号。
```

因此，`r_tho_hat(t)` 和 `r_abd_hat(t)` 不应被写成同一模型里的中间输出。它们分别代表不同参考设定下的独立实验输出。综合呼吸努力估计信号也应放在后续实验中讨论，而不是作为第一阶段必须完成的主输出。

## 3. 研究目标分层

### Level 1：呼吸节律估计

目标：

- 呼吸率轨迹；
- 呼吸周期长度；
- 呼吸主频变化；
- 窗口级呼吸次数。

该层最稳定，适合作为基础目标。

### Level 2：呼吸努力幅值趋势估计

目标：

- 呼吸努力强弱变化；
- 呼吸努力幅值减弱；
- 呼吸努力恢复性增强；
- 周期性呼吸或努力波动；
- 局部努力包络变化。

该层是课题的核心，但应采用相位不敏感监督，重点比较趋势、相对幅值和局部变化。

### Level 3：参考选择与综合呼吸努力构造对比

目标：

- 第一阶段先完成胸带参考下的呼吸努力估计信号；
- 第二阶段在同一流程下完成腹带参考实验；
- 第三阶段比较胸带参考、腹带参考与综合呼吸努力参考之间的关系；
- 研究归一化相加、质量加权、包络融合和学习式融合等综合构造路径。

该层是后续对比实验的一部分，但不应被表述为“单通道 BCG 必然可以精确分离胸腹两路”。胸带估计和腹带估计分别是不同参考设定下的实验结果，而不是同一模型内部的中间表示。

## 4. 信号信息利用原则

### 4.1 BCG 是耦合观测，不是简单加性噪声

枕下压电 BCG 可近似理解为以下成分的混合：

```text
呼吸运动 + 心冲击图 + 体动残余 + 接触状态变化 + 机械传递噪声 + 环境噪声
```

这些成分不是完全独立的。呼吸会调制心冲击幅值、基线、形态和能量分布；体位、枕头接触、头颈位置也会改变传递路径。因此，前处理阶段不宜强行做硬分离。

推荐做法是模型内部软分解：

```text
raw BCG
  -> respiratory latent branch
  -> optional high-frequency context branch
  -> time-frequency branch
  -> optional context branch
  -> gated fusion
  -> respiratory effort estimation signal / RR / envelope
```

### 4.2 呼吸分量

呼吸分量是主输出，但应定义为 BCG 时间轴上的呼吸努力估计信号，而不是 PSG 胸带或腹带的复制品。

第一阶段不需要把所有可能量都设计成模型输出。更合适的做法是只保留一个主输出，其他量优先作为从主输出中计算得到的评价指标。

第一阶段主输出：

- `r_tho_hat(t)`：以 PSG 胸带为参考的呼吸努力估计信号。

从主输出派生或评价：

- `A_tho_hat(t)`：由 `r_tho_hat(t)` 计算得到的呼吸努力包络；
- `RR_tho_hat[k]`：由 `r_tho_hat(t)` 计算得到的窗口级呼吸率；
- `Ridge_tho_hat[k]`：由 `r_tho_hat(t)` 或其时频表示计算得到的主呼吸频率。

可选诊断量：

- `P_breath(t)`：呼吸峰/谷或周期概率，仅在需要解释峰谷定位时使用，不作为第一阶段必要输出。

不同实验中的输出命名可区分为：

- `r_tho_hat(t)`：以 PSG 胸带为参考训练和评价的呼吸努力估计信号；
- `r_abd_hat(t)`：以 PSG 腹带为参考训练和评价的呼吸努力估计信号；
- `r_effort_hat(t)`：以综合呼吸努力参考训练和评价的呼吸努力估计信号。

上述三类输出对应三个独立实验设定，不建议在第一阶段同时作为多头输出。第一阶段默认 `r_hat(t) = r_tho_hat(t)`。

### 4.3 心冲击分量

心冲击不应简单视为噪声，但在当前数据集中也不应被设为必要输入或必要模块。若心冲击成分并不总是清晰可用，模型不应强依赖 J 峰、心搏位置或手工心冲击包络。

高频或心冲击候选频段的 CWT 时频图可以作为优先选择。CWT 幅值图已经包含：

- 高频能量随时间的变化；
- 各频率分量的幅度分布；
- 高频能量是否存在周期性；
- 高频能量是否被呼吸节律调制；
- 高频噪声、接触变化或残余运动的宽带能量特征。

心冲击包络本质上可以看作对心冲击候选频段时频能量的频率方向聚合：

```text
cardiac_envelope(t) = aggregate(CWT(t, f)), f in cardiac band
```

因此，当模型已经输入高频 CWT 幅值图时，额外输入心冲击包络可能是冗余的，也可能在心冲击不可用片段引入错误先验。

推荐定位：

- 必选：原始 BCG、呼吸候选频带和时频图；
- 可选：数据状态或上下文特征；
- 可选：高频/心冲击候选频段 CWT；
- 消融项：心冲击频带包络、J 峰、HR、心搏位置；
- 仅在 ECG 或心搏参考可靠时，才考虑心冲击辅助监督。

### 4.4 时频信息

时频图有价值，但不应作为唯一输入，也不应以“修复时频图再反变换”为主路线。

更合理的使用方式：

- 辅助识别呼吸频谱 ridge；
- 辅助观察呼吸频带能量连续性；
- 辅助区分呼吸、心冲击和残余噪声；
- 用窗口级频谱结构监督模型。

时频特征应与时域原始信号和呼吸候选频带共同输入模型。高频 CWT、数据状态特征可作为辅助上下文；心冲击包络不作为默认必选输入。

### 4.5 接触状态与低频漂移

低频漂移不一定全部是噪声，也可能反映：

- 头颈位置变化；
- 枕头受力变化；
- 体位变化；
- 传感器接触状态变化；
- 机械传递路径变化。

这类信息不适合作为呼吸主输出，但适合作为上下文特征，用于解释片段间差异。

## 5. PSG 参考标签构造

由于同步不精确，应优先构造相位不敏感标签。第一阶段以 PSG 胸带标签为主，腹带标签和综合呼吸努力标签留作后续独立对比实验。

### 5.1 参考构造顺序

第一阶段只使用 PSG 胸带构造主要参考标签，目标是先完整验证 BCG 到呼吸努力估计信号的全流程是否成立。此时不同时引入腹带分支或综合呼吸努力分支，避免模型目标过早复杂化。

建议实验顺序：

```text
A. 胸带参考实验:
   BCG -> r_tho_hat(t)

B. 腹带参考实验:
   BCG -> r_abd_hat(t)

C. 综合呼吸努力参考实验:
   BCG -> r_effort_hat(t)
```

综合呼吸努力参考不应简单固定为胸带和腹带的加性结果。胸带与腹带可能存在幅值差异、质量差异和局部相位差，直接相加有时会造成抵消或错误增强。

可比较以下构造方式：

```text
1. 归一化相加:
   RIPsum = normalized(胸带) + normalized(腹带)

2. 质量加权融合:
   R_effort = w_tho * normalized(胸带) + w_abd * normalized(腹带)

3. 包络融合:
   A_effort = combine(A_tho, A_abd)

4. 呼吸率/频谱层面鲁棒融合:
   RR_ref / Ridge_ref = robust_combine(胸带特征, 腹带特征)
```

其中归一化相加可作为基线，但不应被默认视为唯一正确的综合呼吸努力参考。当前更推荐先分别完成胸带参考和腹带参考实验，再比较不同融合策略对综合呼吸努力估计信号的影响。

注意不要只做全局归一化后丢弃幅值变化。呼吸努力减弱、恢复性增强等目标需要保留相对幅值趋势。

### 5.2 推荐标签

第一阶段可以多提取一些 PSG 胸带派生标签备用，但训练时不必全部用上。标签应分为“默认使用”和“备用分析”，避免把论文主线写成一个过重的多任务系统。

默认使用：

- `RR_tho[k]`：胸带窗口级呼吸率；
- `A_tho[k]`：胸带努力幅值；
- `Ridge_tho[k]` 或 `Spec_tho[k, f]`：二者选择其一作为频谱结构约束，优先使用更稳定的一种。

建议先提取备用，但不默认进入训练：

- `Drop_tho[k]`：胸带相对幅值下降比例；
- `Recovery_tho[k]`：胸带恢复呼吸增强标签；
- `P_peak_tho(t)`：胸带峰/谷弱标签，仅在峰谷定位实验中使用；
- `Spec_tho[k, f]`：若默认已使用 `Ridge_tho[k]`，则作为评价或消融标签。

腹带实验可在相同流程下提取：

- `RR_abd[k]`；
- `A_abd[k]`；
- `Drop_abd[k]`；
- `Spec_abd[k, f]`；
- `Ridge_abd[k]`；
- `Recovery_abd[k]`。

综合呼吸努力实验再构造：

- `RR_effort[k]`；
- `A_effort[k]`；
- `Drop_effort[k]`；
- `Spec_effort[k, f]`；
- `Ridge_effort[k]`；
- `Recovery_effort[k]`。

这些综合呼吸努力标签可以来自胸带/腹带的质量加权融合、鲁棒平均或学习式融合。胸带、腹带和综合呼吸努力标签分别对应独立实验，不作为第一阶段同一模型的多头监督。

这些标签比原始逐点波形更适合弱同步数据。实际建模时应先使用最小标签集合跑通流程，再逐步验证备用标签是否带来稳定增益。

### 5.3 同步关系检查

同步关系检查有意义，但它不应被写成本文的核心方法，也不必在第一阶段建立复杂的 S0/S1/S2 分级体系。从学术研究角度，它更适合作为数据前提审计，用来回答两个问题：

- BCG 与 PSG 胸带是否属于同一时间范围；
- 是否存在明显固定 offset、缓慢 drift 或局部错位，导致逐点波形监督不成立。

推荐做法：

- 在每晚或每个 clean segment 上估计 BCG 低频呼吸候选信号与 PSG 胸带的粗略 lag；
- 统计 lag 分布和最大相关系数；
- 判断窗口级 RR、包络和频谱标签是否仍可用；
- 将结果写入数据说明或实验前置分析。

若检查显示 lag 大体稳定，可在少量消融实验中尝试局部对齐后的波形一致性指标；若 lag 不稳定，则只使用窗口级和趋势级监督。同步检查的结论服务于损失函数选择，而不是形成一个额外的研究分支。

## 6. 推荐模型：WeakSync-RespBCG-Net

### 6.1 输入

```text
x_raw       : 原始 BCG clean 片段
x_resp      : BCG 呼吸候选频带
x_hf_cwt    : 高频/心冲击候选频段 CWT 时频图（可选）
TF          : 全频或呼吸相关 STFT / CWT / SST 时频图
context_pre : rolling energy / STD / derivative energy / clean mask（可选，用于补充分析）
```

其中 `context_pre` 主要用于描述数据状态和进行对比分析，不应把模型主线扩展为复杂的质量兜底系统。

### 6.2 编码器

- 1D Temporal Encoder：学习时域节律、局部形态和长程变化；
- 2D Time-Frequency Encoder：学习呼吸 ridge、频带能量和非平稳变化；
- High-Frequency Context Encoder：可选，用于学习高频/心冲击候选频段的时频上下文；
- Quality/Context Encoder：可选，用于分析接触状态、低频漂移和片段差异对结果的影响。

### 6.3 融合方式

推荐使用 gated fusion 或 cross-attention。融合模块的核心作用是整合时域、时频域和可选上下文信息，而不是构建复杂的质量兜底策略，也不是在第一阶段同时学习胸带/腹带多路输出。

例如：

```text
raw/time branch + TF branch + optional high-frequency TF branch + optional context branch
                    -> gated fusion
                    -> multi-task heads
```

不同参考实验共享同一套主体框架，但分别训练和评价：

```text
胸带参考实验: BCG -> r_tho_hat(t), A_tho_hat(t), RR_tho_hat(t)
腹带参考实验: BCG -> r_abd_hat(t), A_abd_hat(t), RR_abd_hat(t)
综合呼吸努力实验: BCG -> r_effort_hat(t), A_effort_hat(t), RR_effort_hat(t)
```

综合呼吸努力实验可比较多种构造策略，包括归一化相加、质量加权、包络融合和学习式融合。该比较本身可以作为方法研究的一部分，但应放在胸带参考流程跑通之后。

### 6.4 输出

第一阶段默认只要求模型生成一个连续信号：

```text
r_tho_hat(t) : 胸带参考实验的呼吸努力估计信号
```

以下量优先由 `r_tho_hat(t)` 派生，用于损失计算、评价或可视化：

```text
A_tho_hat(t)     : 呼吸努力包络
RR_tho_hat[k]    : 窗口级呼吸率
Ridge_tho_hat[k] : 主呼吸频率
```

不同参考实验中的主输出：

```text
r_tho_hat(t)    : 胸带参考实验的呼吸努力估计信号
r_abd_hat(t)    : 腹带参考实验的呼吸努力估计信号
r_effort_hat(t) : 综合呼吸努力参考实验的呼吸努力估计信号
```

第一阶段最终输出以 `r_tho_hat(t)` 为主。`r_abd_hat(t)` 和 `r_effort_hat(t)` 是后续独立实验输出，不作为第一阶段模型的中间输出或辅助头。

可选诊断输出：

```text
Q_hat(t)          : 质量或不确定性指标（用于对比分析）
HF_context(t)     : 高频上下文指标（用于对比分析）
Card_env_hat(t)   : 心冲击包络（仅作为消融项）
HR_hat(t)         : 心率轨迹（仅在 ECG 或心搏参考可靠时）
J_peak_hat(t)     : 心搏位置概率（仅在 ECG 或心搏参考可靠时）
```

这些可选诊断输出不属于第一阶段默认模型。

## 7. 窗口设计

推荐采用多尺度窗口。

### 短窗：8-16 秒

用于：

- 局部波形；
- 高频局部结构；
- 呼吸峰谷；
- 短时片段特征分析。

### 中窗：30-60 秒

用于：

- 呼吸率估计；
- 呼吸频谱 ridge；
- 包络趋势。

### 长窗：90-180 秒

用于：

- 呼吸努力持续减弱趋势；
- 呼吸努力恢复性增强；
- 周期性呼吸；
- 长程努力变化。

训练时不建议只随机切短窗，否则容易丢失长程趋势和幅值变化。

## 8. 损失函数设计

损失函数不宜一次设计得过多。第一阶段的目标是跑通“BCG -> 胸带参考呼吸努力估计信号”的闭环，因此默认损失应保持小而可解释：

```text
L_main =
  λ1 * L_env
+ λ2 * L_rr
+ λ3 * L_freq
+ λ4 * L_smooth
```

其中：

- `L_env` 约束呼吸努力包络趋势；
- `L_rr` 约束窗口级呼吸率；
- `L_freq` 约束主呼吸频率或呼吸频带谱结构，二者选其一即可；
- `L_smooth` 约束输出信号的基本连续性，避免无意义高频抖动。

第一阶段不建议默认加入过多辅助损失。损失项是否保留，应以消融实验是否稳定提升呼吸率、包络趋势和频谱结构为准。

### 8.1 包络趋势损失

包络趋势是第一阶段最重要的监督目标之一，因为它直接对应呼吸努力强弱变化。

```text
L_env = 1 - Corr(A_tho_hat, A_tho)
```

也可以使用归一化后的 L1/Huber 作为补充，但不建议只依赖逐点幅值误差。若 `Drop_tho` 已经提取，可先作为评价指标或消融标签，不必默认进入主损失。

### 8.2 呼吸率损失

呼吸率损失用于保证模型先学到稳定的呼吸节律：

```text
L_rr = MAE(RR_tho_hat, RR_tho)
```

`RR_tho_hat` 可以由 `r_tho_hat(t)` 的峰谷、频谱主峰或时频 ridge 计算得到。若呼吸率估计本身不可微，可将 RR 作为辅助头输出；但论文表述上仍应强调最终对象是呼吸努力估计信号，而不是单独的 RR 模型。

### 8.3 频率或频谱结构损失

频率/频谱结构损失用于约束呼吸频带位置，避免模型只拟合平滑包络而忽略真实呼吸节律。

二选一即可：

```text
L_freq = MAE(Ridge_tho_hat, Ridge_tho)
```

或：

```text
L_freq = KL(P_hat(f | window), P_tho(f | window))
```

若 `Ridge_tho` 更稳定，优先使用 ridge 损失；若时频结构本身是论文重点，再使用归一化频谱损失。不建议同时默认堆叠 ridge、完整频谱和峰谷损失。

### 8.4 波形一致性损失不作为默认项

类似 shift-invariant waveform loss 的损失在理论上有吸引力，但在当前弱同步条件下不应作为默认训练目标。它依赖局部 lag 估计可靠，一旦 lag 不稳定，模型会被错误惩罚。

因此，波形一致性损失只适合放在消融实验中：

```text
L_wave = min over τ [1 - Corr(r_tho_hat(t), thoracic_ref(t + τ))]
```

使用前提：

- 同步关系检查显示某些片段 lag 稳定；
- 只在这些片段上使用；
- 不作为全文主要结论的依据。

### 8.5 可选扩展项

以下内容不进入第一阶段默认损失，只作为扩展实验或讨论：

- `P_breath(t)` 峰谷弱监督；
- `Drop_tho` 或 `Recovery_tho` 相关损失；
- 不确定性建模；
- 心冲击包络、HR、J 峰等心冲击辅助监督；
- 对比学习预训练；
- GAN 生成约束。

### 8.6 参考通道切换实验中的损失复用

胸带、腹带和综合呼吸努力对应不同参考实验，不应写成同一模型里的多头辅助损失。更合适的做法是保持损失形式一致，只切换参考来源：

```text
实验 A:
  L_tho = L_main(r_tho_hat, thoracic_ref)

实验 B:
  L_abd = L_main(r_abd_hat, abdominal_ref)

实验 C:
  L_effort = L_main(r_effort_hat, effort_ref)
```

当前优先完成实验 A。实验 B 和实验 C 在实验 A 全流程稳定后再开启，用于回答“参考选择是否改变 BCG 呼吸努力估计结果”以及“综合呼吸努力参考如何构造更合理”。

## 9. 当前研究边界

当前研究只关注呼吸努力估计信号提取，不把其他下游任务作为模型输出、训练目标或主评价内容。

第一阶段只保留一个核心输出：

- `r_tho_hat(t)`：胸带参考下的呼吸努力估计信号。

以下结果从 `r_tho_hat(t)` 中派生，用于评价或辅助分析：

- 呼吸率；
- 呼吸努力包络；
- 主呼吸频率或频谱结构。

呼吸峰/谷概率、恢复增强标签、相对幅值下降标签等内容可以先提取备用，但不作为第一阶段必须输出的模型结果。

腹带参考和综合呼吸努力参考作为后续独立对比实验展开，不作为第一阶段模型的中间输出或辅助监督。

其他下游应用可作为后续独立课题或应用扩展，但不纳入本文档定义的当前研究任务。当前任务的核心是证明：BCG 中可以提取出稳定、连续、与 PSG 胸带弱同步参考一致的呼吸努力估计信号。

## 10. 评价指标

### 10.1 呼吸节律指标

- RR MAE；
- RR RMSE；
- breath count error；
- dominant respiratory frequency error。

建议按 30 秒或 60 秒窗口评价。

### 10.2 努力幅值指标

- 包络趋势相关性；
- 幅值下降检出能力；
- 恢复性增强检出能力；
- 局部相对幅值误差；
- 周期性呼吸趋势一致性。

### 10.3 频谱结构指标

- 呼吸主频误差；
- 呼吸频带能量分布一致性；
- 时频 ridge 连续性；
- 多分辨率频谱相似性。

### 10.4 数据覆盖与敏感性分析

由于体动已剔除，可在补充实验或讨论中报告：

- 有效分析时长占比；
- 被剔除片段占比；
- 每夜可用数据比例；
- 不同数据状态分层下的性能。

这些指标用于解释实验样本边界和结果适用范围，不作为本文主模型的核心评价目标。

### 10.5 波形一致性指标

波形一致性只能作为辅助或消融指标，并且需要满足：

- 同步关系检查显示局部 lag 相对稳定；
- 允许最佳 lag 对齐；
- 不作为主要结论依据。

## 11. 实施路线

### Step 1：同步关系检查

对每晚和每个 clean segment：

- 提取 BCG 低频候选呼吸信号；
- 第一阶段构造 PSG 胸带参考；
- 计算 cross-correlation；
- 估计最佳 lag；
- 统计最大相关系数；
- 观察 lag 是否稳定。

输出：

- 每晚 lag 分布；
- 每段 lag 分布；
- 是否存在固定 offset、缓慢 drift 或局部同步失效。

这一步的作用是确认弱同步监督是否成立，并决定是否允许局部波形一致性消融。它不作为论文主贡献，也不需要扩展成复杂的同步可信度分级系统。

### Step 2：构造弱同步标签

第一阶段从 PSG 胸带提取默认训练标签：

- `RR_tho`；
- `A_tho`；
- `Ridge_tho` 或 `Spec_tho`。

同时可提取备用标签：

- `Drop_tho`；
- `Recovery_tho`；
- 峰/谷弱标签；
- 完整呼吸频带谱。

腹带标签和综合呼吸努力标签只在后续独立实验中构造。不要先训练逐点波形回归，也不要在第一阶段把胸带、腹带和综合呼吸努力同时作为多头监督。

### Step 3：建立最小训练闭环

先用最小标签集合跑通：

- BCG 输入片段；
- 胸带参考标签；
- `r_tho_hat(t)` 输出；
- 包络、呼吸率和频率结构评价；
- 片段级可视化检查。

这一步优先保证数据流和训练目标闭环成立，不急于加入传统基线、对比学习、GAN 或复杂心冲击辅助项。

### Step 4：训练弱同步多任务模型

第一阶段模型以胸带参考训练，主输出和派生结果包括：

- `r_tho_hat(t)`；
- `RR_tho_hat`；
- `A_tho_hat`；
- `Ridge_tho_hat`；
- optional context indicators。

此阶段不强求 `r_hat(t)` 与 PSG 逐点对齐。

同时应进行输入消融，判断高频信息是否真的带来增益：

- `raw BCG + respiratory band`；
- 上述输入 + 全频或高频 CWT；
- 上述输入 + 心冲击频带聚合包络；
- 上述输入 + ECG/心搏辅助监督，仅在参考可靠时使用。

若心冲击包络或心搏辅助监督没有稳定提升呼吸率、努力包络、频谱结构或数据状态分层表现，则不应纳入默认模型。

### Step 5：整夜连续评估

在完成片段级建模后，评估模型在整夜数据上的连续输出质量：

```text
r_tho_hat(t)
A_tho_hat(t)
RR_tho_hat(t)
clean mask / segment mapping
```

整夜评价重点是连续性、节律稳定性、包络趋势和有效覆盖范围。数据覆盖与片段剔除比例可作为补充分析。

### Step 6：传统基线与对照实验

传统基线不需要放在第一步之前。更合理的顺序是在胸带参考模型全流程跑通后，再统一实现并比较：

- 低通滤波；
- EWT；
- IEWT；
- CWT/SST ridge tracking；
- Hilbert-based RR；
- 简单包络估计。

传统方法也应使用同一套弱同步评价指标，避免用未对齐波形相关性直接否定其表现。

### Step 7：腹带与综合呼吸努力独立实验

胸带参考流程和基线评价稳定后，再开启两个独立实验：

```text
实验 B：BCG -> r_abd_hat(t)
实验 C：BCG -> r_effort_hat(t)
```

实验 B 使用腹带参考重复同一套训练和评价流程，用于比较胸带参考与腹带参考的差异。实验 C 研究综合呼吸努力参考的构造方式，可比较以下策略：

- 归一化相加；
- 质量加权融合；
- 包络融合；
- 学习式融合模块。

跨实验比较时，应明确不同参考来源造成的差异，而不是把腹带或综合呼吸努力结果解释为胸带模型的中间结果。

## 12. 关键假设与对比讨论

### 12.1 未精确同步导致错误监督

讨论点：

- 逐点 MSE 将同步误差误判为模型误差；
- 模型输出被迫平滑化；
- 呼吸峰谷和幅值下降被削弱。

论文处理：

- 做同步关系检查；
- 主用窗口级、频谱级和包络级监督；
- 波形一致性损失只作为局部 lag 稳定片段上的消融项。

### 12.2 单通道 BCG 无法唯一还原胸腹两路

讨论点：

- 模型根据人群先验猜测胸/腹分量；
- 胸腹反常运动无法可靠恢复。

论文处理：

- 第一阶段以胸带参考呼吸努力估计信号为主；
- 腹带参考和综合呼吸努力参考作为后续独立对比实验；
- 不把胸带/腹带估计写成同一模型内部的分离结果；
- 不确定性分析作为补充讨论，而不是主模型目标。

### 12.3 时频图修复可能丢失相位和幅值意义

讨论点：

- 频谱看似合理，但时域幅值和波形变化不可信；
- 反变换后的时域波形缺乏生理解释。

论文处理：

- 时频图作为辅助输入；
- 输出仍以 RR、包络和呼吸努力估计信号为主；
- 频谱损失不单独作为主目标。

### 12.4 体动剔除导致评价偏差

讨论点：

- 只在容易片段上评价；
- 大量剔除导致整夜可用性被高估。

论文处理：

- 在补充实验中报告有效覆盖率；
- 在讨论中分析数据状态分层对结果的影响。

## 13. 拓展实验与非主线方法

本节内容不属于第一阶段主流程。只有当胸带参考模型已经完成训练、整夜评价和基础对照后，才建议逐项展开。

### 13.1 对比学习预训练

对比学习适合在同步不精确、标注有限的条件下学习稳定表征，但不应放在第一阶段主线中。它可以作为后续扩展，用于比较预训练是否提升呼吸率、包络趋势和频谱结构评价。

可设计正样本：

- 同一 BCG 片段的不同增强版本；
- 同一粗时间窗口内的 BCG 与 PSG 特征；
- 同一受试者同一睡眠阶段的相邻片段；
- 呼吸率或努力包络相近的片段。

### 13.2 GAN 生成约束

GAN 可以被视为一种探索性猜想：用生成约束帮助模型输出更接近真实呼吸努力形态的连续信号。但它不构成本文主方法，也不应放在模型主干或默认损失函数中。

若使用 GAN，应满足：

- 只作为扩展实验或消融实验；
- 只作为小权重弱正则或对比方法；
- 判别器应 conditional on BCG、呼吸节律或上下文特征；
- 必须用呼吸节律、努力包络和频谱结构指标验证，而不能只看波形是否平滑。

### 13.3 高频、心冲击与不确定性建模

高频 CWT、心冲击包络、HR、J 峰、不确定性建模都可以作为扩展实验，但不作为第一阶段必要设计。它们的保留标准应是：是否稳定改善 `r_tho_hat(t)` 的呼吸率、包络趋势或频谱结构评价。

## 14. 最终建议

本课题的核心主张应从“恢复一个漂亮的呼吸波形”转为：

> 模型能够从枕下压电 BCG 中提取胸带参考下的呼吸努力估计信号，稳定估计呼吸节律、努力幅值变化和频谱结构；在体动剔除后的有效片段中，这些输出在窗口级、趋势级和频谱级与 PSG 胸带参考一致，并优于低通、EWT、IEWT 等传统方法。

推荐主路线：

```text
BCG 多视图输入
  -> 弱同步多任务网络
  -> 胸带参考呼吸努力估计信号 / RR / 努力包络
  -> 整夜连续呼吸努力估计
  -> 腹带参考与综合呼吸努力参考的独立对比实验
```

推荐模型重点：

- 不以逐点波形还原为核心；
- 不把时频图修复作为唯一目标；
- 不把心冲击包络、J 峰或 HR 设为必要依赖；
- 不把胸带/腹带估计写成同一模型的中间输出或确定性分离结果；
- 不把体动或低质量片段的兜底处理作为研究主体；
- 优先学习相位不敏感的呼吸节律、幅值趋势和频谱结构。

这一路线更符合当前数据条件，也更容易形成清晰、可信、可评价的研究结果。
